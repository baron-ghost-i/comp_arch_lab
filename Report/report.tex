\chapter{Binary Multiplier Circuits}

\section{Objective}

Design and implement binary multiplier circuits for both signed and unsigned numbers. Verify the functionality of these circuits using test cases for different inputs. Document your design approach and provide a detailed explanation of the multiplier's operation.

\section{Tools Used}
\begin{itemize}
    \item \textbf{Language}: Verilog
    \item \textbf{Compiler}: Vivado 2025.1
\end{itemize}

\section{Design}

For this task, we design a parameterizable single-cycle multiplier which is purely combinational in nature.

Hardware implementations of multipliers which obey algorithms like Booth's algorithm or the classic shift-and-add algorithm require less hardware but more number of clock cycles, which increase linearly with the number of bits used to represent the operands.

Single cycle multipliers, on the other hand, use more hardware to perform faster multiplication. This improves throughput greatly, but also imposes additional restrictions on clock speed due to the circuit's large delay. This can be solved by using pipelining.

For the given task, we implement an array multiplier, which generates partial products by multiplying the multiplicand with each bit of the multiplier separately - an AND operation - and adds these partial products with appropriate shifts to produce the result of multiplication. The shifts, in turn, are implemented through appropriate wire connections only.

\subsection{Design}
\begin{framed}
\begin{minted}{verilog}
`timescale 1ns/1ps

module FA(
    input A, B, Cin,
    output S, Cout
);
    assign S = A^B^Cin;
    assign Cout = (A&B) | (B&Cin) | (Cin&A);
endmodule

module RCA  #(
    parameter N = 4
)(
    input [N-1:0] A, B,
    input Cin,
    output [N-1:0] S,
    output Cout
);
    genvar i;
    wire [N-1:0] C;
    assign Cout = C[N-1];
    FA fa0(.A(A[0]), .B(B[0]), .Cin(Cin), .S(S[0]), .Cout(C[0]));
    generate
        for(i=1; i<N; i=i+1) begin
            FA fa1(
                .A(A[i]),
                .B(B[i]),
                .Cin(C[i-1]),
                .S(S[i]),
                .Cout(C[i])
            );
        end
    endgenerate
endmodule

module Multiplier #(
    parameter N = 4
)(
    input [N-1:0] A, B,
    output [2*N-1:0] P
);
    genvar i, j;
    wire [N-1:0] prod[0:N-1];
    wire [N-1:0] partial[0:N-1];

    generate
        for(i=0; i<N; i=i+1) begin
            assign prod[i] = {N{B[i]}} & A;
        end
        assign partial[0] = {0, prod[0][N-1:1]};
        assign P[0] = prod[0][0];
        
        for(i=1; i<N; i=i+1) begin
            RCA #(.N(N)) rca1(
                .A(partial[i-1]),
                .B(prod[i]),
                .Cin(1'b0),
                .S({partial[i][N-2:0], P[i]}),
                .Cout(partial[i][N-1])
            );
        end
    endgenerate
    assign P[2*N-1:N] = partial[N-1];
endmodule

module signedMultiplier #(
    parameter N = 4
)(
    input [N-1:0] A, B,
    output [2*N-1:0] P
);
    
    genvar i, j;
    wire [N-1:0] Amag, Bmag;
    wire A_cout, B_cout, P_cout, outsign;
    wire [2*N-1:0] Pmag;
    
    // take 2's complement and add 1 if sign bit is set
    RCA #(.N(N)) rca_A(
        .A(A^{N{A[N-1]}}),
        .B('b0),
        .Cin(A[N-1]),
        .S(Amag),
        .Cout(A_cout)
    );
    RCA #(.N(N)) rca_B(
        .A(B^{N{B[N-1]}}),
        .B('b0),
        .Cin(B[N-1]),
        .S(Bmag),
        .Cout(B_cout)
    );
    assign outsign = A[N-1]^B[N-1];
    
    wire [N-1:0] prod[0:N-1];
    wire [N-1:0] partial[0:N-1];

    generate
        for(i=0; i<N; i=i+1) begin
            assign prod[i] = {N{Bmag[i]}} & Amag;
        end
        assign partial[0] = {0, prod[0][N-1:1]};
        assign Pmag[0] = prod[0][0];
        
        for(i=1; i<N; i=i+1) begin
            RCA #(.N(N)) rca1(
                .A(partial[i-1]),
                .B(prod[i]),
                .Cin(1'b0),
                .S({partial[i][N-2:0], Pmag[i]}),
                .Cout(partial[i][N-1])
            );
        end
    endgenerate
    assign Pmag[2*N-1:N] = partial[N-1];
    RCA #(.N(2*N)) rca_out(
        .A(Pmag^{2*N{outsign}}),
        .B('b0),
        .Cin(outsign),
        .S(P),
        .Cout(P_cout)
    );
endmodule
\end{minted}

\section{Discussion}

\begin{enumerate}
    \item The multiplier circuit thus constructed is the single-cycle array multiplier, which is a purely combinational circuit. This, in theory, makes it faster than a multi-cycle multiplier.
    \item The single-cycle multiplier may however not be faster in reality, due to restrictions imposed on clock frequency by the combination of its gate delays. Operations at higher frequencies will inevitably lead to timing violations, leading to incorrect outputs or even metastability within the system.
    \item The architecture implemented can be made to operate at higher frequencies by introducing appropriate pipeline registers, at the cost of more area and hardware.
    
    Pipelining introduces a latency of several clock cycles, depending on its depth, but allows for the same throughput as non-pipelined architecture.
    \item Faster single-cycle multiplier architectures are possible, e.g., Dadda multiplier, Wallace tree multiplier, etc. However, these are not as easily parameterized, and require more hardware.
    \item For small binary operands, ROM can be used instead of combinational logic to precompute and store the results of multiplications.
\end{enumerate}

\section{Conclusion}

In this experiment, a parameterizable array multiplier has been implemented using Verilog HDL, with demonstration for the 4-bit multiplier. Such a multiplier can operate faster than a simple shift-and-add multiplier due to the latter's dependence on the length of binary operands. However, such multipliers inevitably require more area and consume more power due to a larger number of gates required.
